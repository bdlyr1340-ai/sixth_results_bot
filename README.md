# بوت نتائج السادس الإعدادي - تيليغرام + GitHub + Railway

مشروع جاهز لتشغيل بوت يبحث عن نتائج السادس الإعدادي بالاسم الثلاثي أو الرباعي، مع لوحة مالك خاصة لرفع ملفات ZIP لكل محافظة ومتابعة عدد ملفات PDF والطلاب وحالة الفهرسة.

## المزايا

- اختيار المحافظة من أزرار البوت.
- البحث بالاسم الثلاثي أو الرباعي.
- توحيد اختلافات الكتابة العربية مثل `إبراهيم` و`ابراهيم`.
- إظهار أكثر من نتيجة عند تشابه الأسماء مع اسم المدرسة.
- عرض المواد واحدة واحدة: الإسلامية، العربية، الإنكليزية، الأحياء، الرياضيات، الكيمياء، الفيزياء واللغات عند وجودها.
- عرض الرقم الامتحاني، المدرسة، المجموع، المعدل والنتيجة.
- زر دعم اختياري عبر سوبر كي، والرقم الافتراضي: `917392710336`.
- لوحة مالك داخل تيليغرام لا تظهر إلا للحساب الموجود في `OWNER_ID`.
- لوحة ويب محمية بكلمة مرور لرفع ملفات ZIP الكبيرة.
- بطاقة مستقلة لكل محافظة تعرض عدد ملفات PDF والمدارس والطلاب.
- معالجة الملفات في الخلفية مع صفحة حالة تتحدث تلقائيًا.
- استئناف عمليات الفهرسة التي توقفت بسبب إعادة تشغيل الخدمة.
- قاعدة SQLite سريعة، محفوظة على Railway Volume.
- لا تُخزن أسماء البحث التي يكتبها الطلاب؛ تُخزن العدادات العامة فقط.

## لماذا الرفع من لوحة الويب؟

ملفات المحافظات قد تكون أكبر من حد تنزيل ملفات Telegram Bot API. لذلك زر **رفع ملفات المحافظات** في لوحة المالك يفتح صفحة ويب خاصة يمكن من خلالها رفع ZIP كبير مباشرة إلى Railway. يمكن أن يحتوي ZIP على مجلدات داخلية؛ البوت يبحث عن كل ملفات PDF بداخله.

## شكل المشروع

```text
app/
  bot/          معالجات البوت والأزرار ولوحة المالك
  importer/     فك ترميز الخطوط، قراءة PDF، استخراج النتائج
  web/          لوحة رفع ملفات المحافظات
  config.py     متغيرات التشغيل
  database.py   قاعدة البيانات والبحث
scripts/
  check_archive.py   اختبار ملف ZIP من دون إدخاله لقاعدة البيانات
  seed_demo.py       إضافة بيانات تجريبية غير حقيقية
Dockerfile
railway.toml
requirements.txt
```

## 1) إنشاء بوت تيليغرام

1. افتح `@BotFather` في تيليغرام.
2. استخدم `/newbot` وأنشئ البوت.
3. انسخ التوكن وضعه في متغير Railway باسم `BOT_TOKEN`.
4. تحتاج رقم حسابك الرقمي في تيليغرام لوضعه في `OWNER_ID`. يمكن الحصول عليه من أحد بوتات معرفة ID، أو تشغيل المشروع مؤقتًا بقيمة `OWNER_ID=0` ثم استخدام أمر `/id` داخل البوت وتحديث المتغير بعد ذلك.

## 2) تجربة المشروع محليًا

انسخ ملف الإعدادات:

```bash
cp .env.example .env
```

عدّل `.env` ثم شغّل:

```bash
docker compose up --build
```

لوحة المالك:

```text
http://localhost:8000/admin
```

لإضافة بيانات تجريبية غير حقيقية:

```bash
python scripts/seed_demo.py --database data/results.sqlite3
```

ولفحص ملف ZIP الحقيقي قبل رفعه:

```bash
python scripts/check_archive.py /path/to/results.zip --province البصرة
```

## 3) رفع المشروع إلى GitHub

استخدم مستودعًا **خاصًا Private** لأن ملفات النتائج تحتوي بيانات طلاب.

```bash
git init
git add .
git commit -m "Initial sixth results bot"
git branch -M main
git remote add origin YOUR_PRIVATE_GITHUB_REPO
git push -u origin main
```

لا ترفع ملفات ZIP أو PDF أو قاعدة البيانات إلى GitHub. ملف `.gitignore` يمنع ذلك تلقائيًا.

## 4) النشر على Railway

1. أنشئ مشروعًا جديدًا في Railway.
2. اختر **Deploy from GitHub Repo** واربط المستودع الخاص.
3. Railway سيكتشف `Dockerfile` تلقائيًا.
4. أضف **Volume** واربطه بالخدمة على المسار:

```text
/data
```

بدون Volume ستُفقد قاعدة البيانات والملفات عند إعادة النشر.

5. أضف المتغيرات التالية في Railway Variables:

```env
BOT_TOKEN=توكن_البوت
OWNER_ID=رقم_حسابك_الرقمي
DONATION_NUMBER=917392710336
WEB_ADMIN_USERNAME=owner
WEB_ADMIN_PASSWORD=كلمة_مرور_قوية_وطويلة
WEB_SESSION_SECRET=نص_عشوائي_طويل_جدا
DATABASE_PATH=/data/results.sqlite3
STORAGE_DIR=/data/storage
PUBLIC_BASE_URL=https://YOUR-RAILWAY-DOMAIN
PORT=8000
LOG_LEVEL=INFO
```

6. من Networking أنشئ Railway Domain.
7. انسخ الرابط إلى `PUBLIC_BASE_URL` من دون `/` في النهاية، ثم أعد النشر.
8. افتح البوت وأرسل `/admin`. زر **رفع ملفات المحافظات** سيفتح لوحة الويب.

## 5) رفع ملفات محافظة

من لوحة الويب:

1. افتح بطاقة المحافظة المطلوبة، مثل بغداد أو البصرة.
2. اضغط **رفع ملف ZIP لهذه المحافظة**.
3. اختر الفرع والسنة والدور.
4. اترك خيار الاستبدال محددًا عندما تريد تحديث نفس المحافظة والسنة والدور.
5. ارفع الملف.
6. ستظهر صفحة تقدم بعدد ملفات PDF المفحوصة وعدد الطلاب والتنبيهات.

بعد اكتمال الفهرسة، تظهر المحافظة مباشرة ضمن أزرار البحث في البوت.

## طريقة قراءة ملفات النتائج

المستورد يقوم بالخطوات الآتية تلقائيًا:

1. فك ZIP بأمان ومنع مسارات ZIP الضارة.
2. العثور على كل ملفات PDF حتى داخل المجلدات الفرعية.
3. استخراج الخط العربي المضمّن من ملف PDF في الذاكرة، من دون تخزين أو نشر ملفات الخطوط.
4. فك الحروف المشفرة وتصحيح الأرقام التي تظهر معكوسة بسبب اتجاه الكتابة.
5. استخراج بيانات المدرسة والطالب ودرجات المواد والنتيجة والصفحة الأصلية.
6. توحيد الاسم العربي وبناء فهرس بحث سريع.

## أوامر البوت

- `/start` القائمة الرئيسية.
- `/id` إظهار Telegram ID للحساب.
- `/admin` لوحة المالك، ولا تعمل إلا مع `OWNER_ID`.

## ملاحظات مهمة

- اجعل مستودع GitHub خاصًا.
- اختر كلمة مرور قوية للوحة الويب.
- لا تشارك `BOT_TOKEN` أو `WEB_SESSION_SECRET`.
- نتيجة الطالب تُعرض كما وردت في الملف؛ رموز مثل `م` أو `غ` لا يتم تحويلها إلى درجات.
- راجع سجل التنبيهات بعد كل رفع. إذا فشل ملف PDF معين سيظهر اسمه في صفحة العملية.
- عند تغير تصميم ملفات وزارة التربية مستقبلًا قد يحتاج محلل الصفوف إلى تحديث بسيط، لكن فك الخط واستخراج الخط المضمّن يعملان تلقائيًا مع القالب الحالي.
